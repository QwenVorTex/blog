## 色彩这件事，真不能只靠感觉

我一开始想做这个博客的时候，脑子里最先冒出来的其实不是排版，而是颜色。

我很清楚自己不想做那种特别稳妥的灰白蓝组合，我想要更直接一点的视觉冲击。所以才会有现在这套偏亮黄、纯黑、钴蓝、猩红的搭配。

但问题也很快来了：颜色一旦放大胆，界面就特别容易脏。

不是说颜色本身脏，而是如果层级没有控制住，整个页面会开始互相抢注意力。看到最后，哪里都亮，哪里都吵。

所以我后来慢慢接受了一件事：色彩不是装饰，至少在这个博客里不是。它更像是一套约束系统。

## 我最后留下来的几种颜色

这套博客现在大概就是下面几个核心颜色：

| 色彩层级 | HEX       | 我主要拿它干什么 |
| -------- | --------- | ---------------- |
| 空间基底 | `#F9F8F4` | 做大面积阅读背景 |
| 绝对边界 | `#000000` | 边框、阴影、结构切分 |
| 复古亮黄 | `#FFCB2F` | 主要强调和视觉起点 |
| 猩红撞色 | `#E32A2A` | 高优先级动作和强提醒 |
| 钴蓝高亮 | `#1578EA` | 链接、信息类强调 |

看起来挺简单，但其实我后来很多决定都是围着这几种颜色在做减法，而不是加法。

## 60-30-10 这个老东西还是有用

我以前其实对这种“设计法则”有点半信半疑，总觉得很多时候讲得像口诀。

但真到自己做站的时候，还是会发现它有用。60-30-10 说白了就是帮你控制冲动：

- **60%** 给背景和阅读空间
- **30%** 给主要视觉块
- **10%** 给最需要跳出来的动作

如果不这么克制，最后最常见的结果就是：你以为自己做的是大胆，实际做出来的是乱。

## 为什么我一直坚持用纯黑边框

这个博客里几乎到处都能看到纯黑边框，不只是因为它“看起来够狠”，更重要的是它真的有效。

```css
.btn {
  border: 4px solid #000000;
  box-shadow: 8px 8px 0px 0px #000000;
}
```

尤其在大面积撞色环境里，纯黑边框的作用非常直接：它把元素和背景切开。

你不用猜这里是不是按钮，不用猜这块是不是卡片，也不用担心某个高亮色一旦铺开就把层级冲散。黑边框像一道很粗暴但很好用的结构线，直接把版面稳住。

## 无障碍不是后来补的

我不想做那种“看起来很强，实际上读起来很累”的界面，所以对比度这件事从一开始就在我的考虑里。

像亮黄这种颜色，视觉上很讨喜，但拿来做大面积文本背景时如果不配纯黑字，立刻就会出问题。再比如蓝色和红色，拿来做强调没问题，但如果一上来就把它们当正文背景，十有八九会翻车。

所以我最后的做法其实很朴素：大胆的颜色可以用，但必须服从清晰度。

## 说到底，这还是一套秩序

很多人会把新粗野主义理解成“怎么炸怎么来”，但我自己做下来反而觉得，它比很多看起来很干净的风格更需要克制。

因为你手里拿的是更锋利的东西。颜色、边框、阴影、动效，任何一个用过量了都会马上出事。

所以现在这套调色板对我来说，不是为了证明我能把颜色配得多猛，而是为了让这个博客即使很跳，也还像一个能长期阅读、长期维护的地方。
